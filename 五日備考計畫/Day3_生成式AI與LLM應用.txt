----------------------------------------------------------------
2. 提示工程 Prompt Engineering
----------------------------------------------------------------
* Zero-shot: 直接問
* Few-shot: 給範例後問
* Chain-of-Thought (CoT): 讓模型逐步推理
* Prompt 攻擊型態 (★必考):
    - Prompt Injection 提示注入: 使用者輸入操控模型偏離任務
    - Prompt Leakage 提示洩漏: 模型暴露不該透露的系統提示
    - Jailbreak 越獄: 繞過安全限制
* 對策: 輸入過濾、輸出檢查、權限分層

----------------------------------------------------------------
3. RAG 與微調 (★關鍵比較)
----------------------------------------------------------------
* RAG (Retrieval-Augmented Generation):
    - 檢索外部知識 + 生成
    - 優點: 不用重訓、即時更新、可追溯來源
    - 適合: 企業內部文件問答
* Fine-tuning 微調:
    - 修改模型參數
    - 適合: 改變模型風格、特定領域語氣
* RAG vs Fine-tune: RAG 快 (無需重訓), Fine-tune 深
* LoRA: 高效微調, 只訓練少量參數
* 蒸餾: 大模型壓成小模型
* 幻覺 Hallucination: LLM 自信地編造錯誤資訊
    對策: RAG、Temperature 調低、人工審核

----------------------------------------------------------------
4. No-code / Low-code 平台 (★ 約 5~10 題/場)
----------------------------------------------------------------
* No-code: 無需程式碼 (Make, Zapier, Bubble)
* Low-code: 少量程式 (Power Apps, OutSystems)
* 優勢: 開發快、降低門檻、原型驗證
* 限制:
    - 客製化彈性受限
    - 大規模/高效能不適合
    - 廠商鎖定 (Vendor Lock-in)
    - 複雜邏輯難實作
* 適用對象: 業務人員、PM、原型開發

----------------------------------------------------------------
5. AI Agent 與多模態
----------------------------------------------------------------
* AI Agent 核心能力:
    - 感知 Perception
    - 規劃 Planning
    - 工具呼叫 Tool Use
    - 記憶 Memory
    - 自主執行
    - ★ 不包含: 發明新技術
* AI 環境適應能力 = 泛化能力 Generalization
* 多模態: 同時處理文字 / 圖像 / 語音 / 影片
* CLIP: 將圖片與文字映射到同一嵌入空間
